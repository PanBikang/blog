                                  Probabilistic language modeling(LM)

                                  Goal: compute the probability of a sentence.

                                  P(w1,w2,⋯,wn)

                                  any modeling complete this goal can be treated as language modeling.

                                  Applications:

                                  Machine Translation: P(high winds tonight)>P(large winds tonight)

                                  Spell correction: P(about fifteen minutes from)>P(about fifteen minuets from)

                                  Speech Recognition: P(I saw a van)>P(eyes awe of an)

                                  Chinese IME: P(自然语言)>P(孜然鱼雁)

                                  A trivial LM

                                  Estimate sentence probability from its frequency in a corpus.

                                  It doesn't work!

                                  Applying chain rule

                                  The probability of every word depends on the words before it.

                                  P(w1,w2,⋯,wn)=∏iP(wi|w1,w2,⋯,wi−1)

                                  It doesn't work again!

                                  Three methods to be introduced: n-Gram LM, RNN, Transformers

                                  Problem: unknown words

                                  How to deal with some words we haven't ever seen, which is called OOV(out of vocabulary).

                                  Solution: Create an unknown word token <UNK>

                                  Substitute <UNK> with OOV words and treat them as the same word.

                                  Alternative Solution: Build a language model at the character level.

                                  Evaluation

                                  Extrinsic evaluation

                                  Use downstream tasks performance to evaluate the model which is time-consuming and different between different tasks.

                                  More often: Intrinsic evaluation-perplexity

                                  The average log-probability per word of x¯1:m is

                                  l=−1M∑i=1mlog2⁡p(x¯i)

                                  where M=∑i=1m|x¯i|. This is the average number of bits required to encode each word.

                                  we take 2l as perplexity and the lower is better. The perplexity is used to evaluate a language model.

                                  Caution: never compare the perplexities of LMs with different vocabularies.

                                  N-gram models

                                  Markov Assumption

                                  P(wi|w1,w2,⋯,wi−1)≃P(wi|wi−k,⋯,wi−1)

                                  Simplest case: Unigram model

                                  P(w1,w2,⋯,wn)≃∏iP(wi)

                                  Note that this model is similar to naive Bayes.

                                  In principle, this is an insufficient model of language because the language has long-distance dependencies. But n-gram models are surprisingly good for LM.

                                  Estimating bigram probabilities

                                  We use MLE to estimate the probabilities. This method has some problems: data sparseness. Some data doesn't exist in the training set exists in the test set.

                                  A simple method to tackle this problem is to add λ>0 to every count(pseudo count) before normalizing.

                                  Backoff

                                  We use n-gram as a normal method and use (n-1)-gram otherwise.

                                  Interpolation

                                  mix unigram, bigram, trigram, ⋯

                                  example: Linear Interpolation

                                  Two different interpolations: Simple interpolation and lambdas condition on context

                                  P(wn|wn−2wn−1)=λ1P(wn|wn−2wn−1)+λ2P(wn|wn−1)+λ3P(wn)

                                  P(wn|wn−2wn−1)=λ1(wn−2wn−1)P(wn|wn−2wn−1)+λ2(wn−2wn−1)P(wn|wn−1)+λ3(wn−2wn−1)P(wn)

                                  The text generated by n-gram is surprisingly grammatical but incoherent! We need to consider more than three words at a time if we want to model language well. But increasing n worsens the sparsity problem and increases model size exponentially.

                                  Recurrent neural networks

                                  A fixed window neural language model

                                  w(1),x(2),x(3),x(4) are words/one-hot vectors and word embedding e=[e(1);e(2);e(3);e(4)]. The hidden layer is calculated by h=f(We+b1) and the output distribution is calculated by y^=softmax(Uh+b2)

                                  Improvements over n-gram LM:

                                  • No sparsity problem
                                  • Don't need to store all observed n-grams and the complexity of this model grows linearly or in polynomial complexity.

                                  Remaining problems:

                                  • fixed window is too small
                                  • enlarging window enlarges W

                                  RNN LM

                                  x(t) are words, e(t)=Ex(t) are word embedding, h(t)=σ(Whh(t−1)+Wee(t)+b1) is hidden layer and y^(t)=softmax(Uh(t)+b2)

                                  Advantages:

                                  • no longer an n-gram model
                                  • moderate model size

                                  We train RNN LM with maximum likelihood estimation and optimize the parameters with stochastic gradient descent

                                  Vanishing gradient problem

                                  When the gradient in RNN is all small, the gradient signal gets smaller and smaller as it backpropagates further.

                                  Why is vanishing gradient a problem?

                                  Gradient signal far away from is lost because it's smaller than gradient signal from close-by. So model weights are updated only with respect to near effects but not long-term effects.

                                  Exploding gradient problem:

                                  When these are large, the gradient signal gets larger and larger as it backpropagates further.

                                  Solution: gradient clipping

                                  If the norm of the gradient is greater than some threshold, scale it down before applying SGD update.
